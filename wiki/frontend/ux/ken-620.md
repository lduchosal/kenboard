### Comportement obtenu

- Carte home page "Tâches par board" : camembert où chaque slice = une catégorie, taille = somme `task_counts_by_project.total` (toutes statuts confondus) sur les projets de la catégorie.
- Catégories à zéro tâche : exclues (pas de slice, pas de légende).
- Couleur slice = `cat.color` (cohérent avec le reste du dashboard — dots, burndown, kanban).
- Légende : pastille couleur, nom (lien vers la cat), count en gras, pourcentage en dimmed.
- Tout est server-side : aucun JS ajouté, aucun lib de charting (cohérent avec activity.html, taskers, burndown).
- Page `/cat/<id>.html` inchangée — elle utilise toujours `wiki_sections_by_project.html` (bar chart par projet de la catégorie, #572) qui partage les CSS `.wiki-cat-*` / `.wiki-bar-*` conservés intactes.

### Limitation

- Rendu visuel réel pas vérifié (la home requiert login + base avec données → pas reproductible en CLI). Le test client Flask couvre la structure HTML/SVG (paths, légende, titres). Le visuel sera observable post-publish sur kenboard.2113.ch.

### Pourquoi cette forme

- Slices pré-calculés en Python : pas de dépendance JS, cohérent avec les autres widgets home (activity, taskers, burndown). Un tooltip natif via `<svg><title>` suffit (texte au hover du slice).
- Single-slice traité comme cercle complet (deux arcs 180°) sinon `M cx cy L ... A ... Z` dégénère en point quand sweep == 2π.
- Le SVG est cliquable slice par slice (lien vers `/cat/<id>.html`) + chaque entrée de légende est aussi un lien — deux affordances pour le même geste, comme les cat-cards de la home.
