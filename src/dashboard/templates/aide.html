Aide — utiliser le kenboard

Bots Le ken pour les agents

Piloter le board depuis un agent LLM (Claude, GPT…) ou un terminal, via la CLI ken.

1. Installer

python3 -m venv .venv && source .venv/bin/activate
pip install kenboard

2. Configurer

Le plus simple : sur une catégorie, cliquez Copy onboard link et collez l'URL à votre agent — il récupère un runbook complet et s'auto-configure. Sinon, créez un fichier .ken (mode 0600) :

project_id=<UUID du projet>
base_url={{ base }}
api_token=<clé API depuis /admin/keys>

Générez la clé sur /admin/keys (scope read ou write sur le projet).

3. Travailler

ken list --who Claude --status todo
ken show <id>
ken add "MODULE / Titre" --desc "…" --who Claude --status todo
ken move <id> --to doing
ken move <id> --to review

Workflow : todo → doing → review → done. L'agent gère jusqu'à review ; seul l'humain finalise en done.

Extension Le ken pour le navigateur

Capturer l'onglet courant comme tâche en un raccourci, avec capture d'écran optionnelle.

1. Télécharger

Chaque release attache l'extension : github.com/lduchosal/kenboard/releases → kenboard-extension-<version>.zip. Dézippez-la localement.

2. Installer

Chrome / Edge / Brave : chrome://extensions/ → activer le Mode développeur → Charger l'extension non empaquetée → choisir le dossier dézippé.

Firefox (temporaire) : about:debugging#/runtime/this-firefox → Charger un module temporaire → manifest.json.

Firefox (permanent) : installez le .xpi signé via about:addons → ⚙ → Installer un module depuis un fichier. Il survit aux redémarrages.

3. Configurer

Cliquez l'icône de l'extension → Réglages, puis collez votre Copy onboard link dans le champ Onboarding link : l'URL de base, le projet et le token se remplissent automatiquement. Cliquez Test connection puis Save.

4. Utiliser

Raccourci Ctrl+Shift+K (macOS : Cmd+Shift+K), ou l'icône de la barre d'outils. Le titre et l'URL de la page sont pré-remplis ; éditez, puis Save. La tâche atterrit en todo dans le projet configuré.
